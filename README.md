
# MechTeam Design System - English

## Type
Product Design System

## Name
MechTeam Module System

## Idea
Use the same (where applicable) components for all brand entities.  
Target an easy-to-use UI and deliver a clean and accessible UX.

---

## Components
- Simplistic, bold components.  
- All brand entities must be easy to understand and capture the eye of potential customers.  
- Use transparent objects over interesting gameplay backgrounds.  
- Showcase gameplay at all times.  
- Use transparency when content does not allow for traditional foreground images.

---

## CSS Methodology: BEM + Utility
- Use repeating elements (main classes) which can be modified with **modifier classes**.  
- Each modifier class can be used on all elements (main classes).  
- Each main class works with each modifier.  

> This differs from OOCSS, where modifier classes are unique to each main class.

Each component must exist within a **container**.  
Each component is defined by:  
- Its container  
- Its main class  
- Its additional modifier classes (if any)

---

## UX Rules
- Simple and easy to understand UI.  
- Potential players must always experience the feeling of the game just by looking at any brand entity.  
- UX of all brand entities must be consistent.

---

## Visual Styles
Experimental style combining:  
- **Glassmorphism** (due to transparency use)  
- **Soft UI** (due to color palette)  
- **Urban Retro Sci-Fi Aesthetic**  
- **Flat UI**  

> Style chosen to mimic in-game UI.

---

## Дизайн система за бранда MechTeam - Български

**Тип:** Продуктова дизайн система  
**Име:** Модулна система MechTeam  

**Идея:** Използване на едни и същи (където е приложимо) компоненти за всички бранд елементи.  
Целта е лесен за използване интерфейс и чисто и достъпно потребителско изживяване (UX).

---

## Компоненти
- Семпли компоненти.  
- Всички бранд елементи трябва да бъдат лесни за разбиране и да привличат вниманието на потенциалните играчи.  
- Използване на прозрачни обекти върху интересни геймплей фонове.  
- Геймплеят трябва да се показва постоянно.  
- Прозрачността се използва в случаите, когато съдържанието не позволява традиционни изображения на преден план.

---

## CSS методология: BEM + Utility
- Използване на повтарящи се елементи (основни класове), които могат да се модифицират чрез **модификаторни класове**.  
- Всеки модификаторен клас може да се използва върху всички елементи (основни класове).  
- Всеки основен клас работи с всеки модификатор.  

> Това се различава от OOCSS, тъй като при OOCSS модификаторните класове са уникални за всеки основен клас.

Всеки компонент трябва да съществува в **контейнер**.  
Всеки компонент се дефинира чрез:  
- неговия контейнер  
- неговия основен клас  
- неговите допълнителни модификаторни класове (ако има)

---

## UX правила
- Семпъл и лесен за разбиране интерфейс.  
- Потенциалните играчи трябва винаги да усещат атмосферата на играта само като погледнат който и да е бранд елемент.  
- UX на всички бранд елементи трябва да бъде последователен и консистентен.

---

## Визуални стилове
Експериментален стил, комбиниращ:  
- **Glassmorphism** (поради използването на прозрачност)  
- **Soft UI** (поради цветовата палитра)  
- **Urban Retro Sci-Fi естетика**  
- **Flat UI**  

> Стилът е избран с цел да наподобява интерфейса в самата игра.
